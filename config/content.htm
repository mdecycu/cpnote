登入後, 學習如何利用 chmod 針對特定目錄或檔案設定存取權, 例如: chmod 711 /home/your_account, 其中的 7 為 r = read 以 4 代表, w = write 以 2 代表, x = execute 以 1 代表, 且 Linux 中每一個檔案或目錄都用 10 個字元代表其屬性, 第一個字元若為 d = directory 表示該名稱為目錄, 隨後的 2-4 字元代表 user 對該檔案或目錄的 rwx 存取權, 5-7 字元則代表 group = 同一群組用戶對該檔案或目錄的存取權, 8-10 字元則代表 others = 非用戶非同一群組的其他身分使用者對該檔案或目錄的存取權.

因此, chmod 711 /home/your_account, 表示要將帳號對應目錄設為 user 可以 rwx, 而同一群組及其他用戶則只能 x = execute 執行.

若使用 chmod 755 /home/your_account/public_html 進行設定, 表示該目錄中的所有內容, user 可以 rwx, 而同一群組及其他用戶則只能 r = read 以 4 代表, 然後加上 x = execute 以 1 代表, 也就是 4 + 1 = 5 的 r-x 權限, 即可以讀取與執行的權限.

有關 s1511 伺服器:

s1511.cycu.org 是一台在 Windows 環境中使用 Virtualbox 建立的虛擬主機, 其操作系統為 Ubuntu 22.04 LTS (Ubuntu 各版次的生命週期), 而 Ubuntu 則是 Linux 操作系統中的一個分支.

因為這台主機已經配置能夠讓 cp2023 執行動態編輯程式的環境, 所以只要使用者能夠利用 OpenSSH 協定, 連結 s1511 與 github.com, 就可以利用 ssh 協定將倉儲 git clone 到 s1511 主機中.

所謂"連結 s1511 與 github.com", 就是指使用者已經將帳號下的 .ssh/id_rsa.pub 的內容送到 Github 帳號 settings 對應的 SSH and GPG keys 中進行登錄, 且 .ssh/config 已經設定從 IPv6 以 ssh 協定透過代理主機連線到 github.com 的內容.

.ssh 目錄中的 config, 將 SSH session 名稱設為 github.com:

Host github.com
User git
Hostname github.com
ProxyCommand /usr/bin/ncat --proxy p4.cycu.org:3128 --proxy-type http %h %p

所謂"這台主機已經配置能夠讓 cp2023 執行動態編輯程式的環境", 指的是這台主機已經在 Python3 環境安裝 cp2023 啟動所需的模組.

也就是已經透過管理者執行 pip install flask flask_cors bs4 lxml pelican markdown pyopenssl

而且透過 WSGI 讓使用者只要執行 cp2023 倉儲中的 server.py, 就能開啟動態網頁編輯器.

但多人存在同一台伺服器使用 WSGI 服務時, 則必須各用戶使用不同的內部與外部埠號, 因此只要各用戶使用分配到的內部 port 與外部 port 進行設定, 各用戶就可以在 cp2023 倉儲啟動編輯後, 登入修改內建的 admin 管理者密碼, 並且以 chmod o-r config/config 指令, 取消 others (也就是 Linux 操作系統下的所有帳號) 的 read 權限, 以保全管理者帳號的編碼. (在 Replit 是透過  Secrets 中的 config 變數設定進行保全).

建議學習計算機程式課程的心態與方法:

第一點: 由於計算機程式是透過指令或特定敘述與電腦軟硬體進行互動, 且最直接的程式內容都使用英文與電腦進行溝通, 因此在學習計算機程式的過程, 儘量使用全英文模式, 當電腦回應英文或解題過程碰到英文教材, 一定要仔細閱讀, 充分了解各英文單字與文句段落的意涵, 之後才能靈活運用.

第二點: 所謂"儘量使用全英文模式"的意思為: 登入系統的帳號使用英文命名、電腦中目錄與檔案的命名使用英文、程式中的註解與變數或函式命名也儘量使用有意義的英文 (可以透過線上翻譯系統, 將中文語句轉為英文)、改版提交敘述也儘量使用英文描述、網頁的標題也儘量使用英文.

第三點: 閱讀英文教材並且不斷提升自我英文在聽、說、讀、寫的能力的方法是: 不要排拒英文教材, 因為特定系統的完整說明文件, 通常是用英文編寫. 當電腦回應英文敘述, 或者在閱讀英文教材時, 可以利用 AI 系統的輔助, 加快對於內容的深入了解, 並可從中增加自我英文字彙與閱讀能力. (可以利用網路上許多優質的計算機程式相關英文教學影片輔助學習).

第四點: 電腦輔助設計室的電腦已經加裝還原卡, 為了快速客製化電腦以提高使用效率, 必須使用隨身碟中的批次檔 (batch)、登錄檔 (registry) 或 Powershell 指令檔進行設定, 例如: 網路環境、Putty 工作階段、程式編譯環境、程式解譯環境、關閉額外的電腦服務及負載等.

第五點: C、Javascript 與 Python 是機械設計工程師目前經常會碰到的計算機程式語言, 初學者可以按照此一順序進行學習. C 經常被用來控制電腦硬體, Javascript 內建在每一個瀏覽器中, 而 Python 則是從精簡解題到跨足 AI 領域的最佳解譯系統. 之後若需要涉足較大型電腦套件, 可以再深入研究 C++、C#、Java、Rust 與 Mojo 等計算機程式語言的應用.

第六點: 無論學習那一種計算機程式語言, 所有的程式師都是從"Hello World!"簡單列印開始, 透過自學與網路協同環境, 不斷強化自我的程式解題能力. 建議所有學員必得要將所碰到的程式範例一字字打字開始, 針對編譯、連結與執行, 或者解譯後執行的每一個細節充分理解, 之後才能有機會在碰到產品開發過程中棘手問題時, 享受利用計算機程式解題的方便與效能.

2. 接下來, 在 Two-factor methods 下點選 Authenticator app 右方的 Add (若要變更手機所使用的 2FA token 產生程式, 則點選 Edit), 視窗中將會出現一組二維條碼, 使用者可以利用手機開啟 Google Authenticator 或 Authy 中的 account add 功能開啟相機拍二維條碼後, 將 Github 帳號登錄至手機認證套件中.

3. 一旦上述 Github 帳號在手機中的 2FA 認證設定完成, 使用者可以將前述內建為 Github Mobile 的 Preferred 2FA methods 改為 Authenticator app, 之後再登入 Github, 除了必須輸入對應的帳號密碼之外, 還需要輸入在特定時段內輸入手機或手錶中對應帳號所顯示的六個數字 (即 2FA token), 才能登入 Github.

4. 至於手機中的 Authenticator 也可以再利用 Pin 密碼、指紋或人臉辨識流程加入安全保護, 一旦手機或手錶毀損, 就可以藉以登入 Github 帳號修改設定.

// https://en.wikipedia.org/wiki/Flag_of_the_Republic_of_China
// 內政部國旗參考資料: https://www.moi.gov.tw/cp.aspx?n=10621
// cc roc_flag_in_gd.c -lgd -lm to link with gd and math library
// https://www.rapidtables.com/web/color/RGB_Color.html
// 幾何形狀著色與繪圖練習
// 以下 gd 繪圖程式嘗試畫出 ROC 國旗, 請根據下列程式內容完成後續的國旗繪圖
#include <stdio.h>
#include <gd.h>
#include <math.h>

void draw_roc_flag(gdImagePtr img);
void draw_white_sun(gdImagePtr img, int x, int y, int size, int color);

int main() {
    // width 3: height 2
    int width = 1200;
    // 國旗長寬比為 3:2
    int height = (int)(width*2.0 / 3.0);

    gdImagePtr img = gdImageCreateTrueColor(width, height);
    gdImageAlphaBlending(img, 0);

    draw_roc_flag(img);

    FILE *outputFile = fopen("./../images/roc_flag_in_gd.png", "wb");
    if (outputFile == NULL) {
        fprintf(stderr, "Error opening the output file.\n");
        return 1;
    }
    gdImagePngEx(img, outputFile, 9);
    fclose(outputFile);
    gdImageDestroy(img);
    return 0;
}

void draw_roc_flag(gdImagePtr img) {
    int width = gdImageSX(img);
    int height = gdImageSY(img);
    int red, white, blue;
    // 白日位於青天面積正中央, 因此中心點座標為長寬各 1/4 處
    int center_x = (int)(width/4);
    int center_y = (int)(height/4);
    // gdImageFilledEllipse 需以長寬方向的 diameter 作圖
    // 由於中央白日圓形的半徑為青天寬度的 1/8
    // 因此中央白日圓形的直徑為青天寬度的 1/4, 也就是國旗寬度的 1/8
    // 而且白日十二道光芒的外圍圓形其半徑也是國旗寬度的1/8
    int sun_radius = (int)(width/8);
    // 中央白日圓形的直徑等於十二道光芒外圍圓形的半徑
    int white_circle_dia = sun_radius;
    // 中央藍色圓形半徑為中央白日的 1又 2/15
    int blue_circle_dia = white_circle_dia +  white_circle_dia*2/15;
    // 根據 https://www.moi.gov.tw/cp.aspx?n=10621 訂定國旗三種顏色值
    red = gdImageColorAllocate(img, 255, 0, 0); // 紅色
    white = gdImageColorAllocate(img, 255, 255, 255); // 白色
    blue = gdImageColorAllocate(img, 0, 0, 149); // 藍色
    // 根據畫布大小塗上紅色長方形區域
    gdImageFilledRectangle(img, 0, 0, width, height, red);
    // 青天面積為整面國旗的 1/4, 也是採用長方形塗色
    gdImageFilledRectangle(img, 0, 0, (int)(width/2.0), (int)(height/2.0), blue);
    // 先設法以填色畫出六個白色堆疊菱形
    draw_white_sun(img, center_x, center_y, sun_radius, white);
    // 利用一個藍色大圓與白色小圓畫出藍色環狀
    gdImageFilledEllipse(img, center_x, center_y, blue_circle_dia, blue_circle_dia, blue);
    gdImageFilledEllipse(img, center_x, center_y, white_circle_dia, white_circle_dia, white);
  
}

void draw_white_sun(gdImagePtr img, int center_x, int center_y, int sun_radius, int color) {
    // M_PI 大小定義於 math.h 標頭檔中, 因為三角函數中採用徑度為角度單位
    // 因此定義將角度轉為徑度的轉換變數為 deg, 角度值乘上 deg 就可轉為徑度
    float deg = M_PI/180;
    // 根據十二道光芒的每一尖角的角度為 15 度, 求出其對應直角三角形的另一角度為 75 度
    // 求出十二道光芒中任一菱形的 small radius, 也就是菱形的另一個對應小圓的半徑大小
    float sr = sun_radius/tan(75*deg);
    int ax, ay, bx, by, dx, dy, ex, ey;
    gdPoint points[4];
    /* 在塗上十二道光芒中的單一菱形區域之前, 先以座標點畫線測試是否正確
    ax = center_x;
    ay = center_y - sun_radius;
    bx = center_x - sun_radius*tan(15*deg);
    by = center_y;
    ex = center_x;
    ey = center_y + sun_radius;
    dx = center_x + sun_radius*tan(15*deg);
    dy = center_y;
    // AB
    gdImageLine(img, ax, ay, bx, by, color);
    // BE
    gdImageLine(img, bx, by, ex, ey, color);
    // ED
    gdImageLine(img, ex, ey, dx, dy, color);
    // DA
    gdImageLine(img, dx, dy, ax, ay, color);
    */
    ax = center_x;
    ay = center_y - sun_radius;
    bx = center_x - sun_radius*tan(15*deg);
    by = center_y;
    ex = center_x;
    ey = center_y + sun_radius;
    dx = center_x + sun_radius*tan(15*deg);
    dy = center_y;
    // 確定單一菱形區域的塗色正確後, 利用迴圈每次轉動 30 度, 總共轉六次即可塗上十二道光芒區域
    for (int i=1;i<=6;i++){
    // A
    points[0].x = ax+sun_radius*sin(30*deg*i);
    points[0].y = ay+sun_radius-sun_radius*cos(30*deg*i);
    // B
    points[1].x = bx+sr-sr*cos(30*deg*i);
    points[1].y = by-sr*sin(30*deg*i);
    // E
    points[2].x = ex-sun_radius*sin(30*deg*i);
    points[2].y = ey-(sun_radius-sun_radius*cos(30*deg*i));
    // D
    points[3].x = dx-(sr-sr*cos(30*deg*i));
    points[3].y = dy+sr*sin(30*deg*i);
    // 對菱形區域範圍塗色
    gdImageFilledPolygon(img, points, 4, color);
    // 在菱形區域外圍畫線, 明確界定菱形範圍
    gdImagePolygon(img, points, 4, color);
    }
}

#include <stdio.h>
#include <string.h>

int main() {
    // Open the user file for reading
    FILE* user_file = fopen("2b_user_list.txt", "r");
    if (user_file == NULL) {
        perror("Error opening user file");
        return 1;
    }

    char line[100]; // Assuming a maximum line length of 100 characters

    char valid_users[100][20]; // Assuming a maximum of 100 valid users with a length of 20 characters each
    int valid_user_count = 0;

    // Read and store the student numbers from the user file
    while (fgets(line, sizeof(line), user_file)) {
        line[strcspn(line, "\n")] = '\0'; // Remove the newline character
        strcpy(valid_users[valid_user_count], line);
        valid_user_count++;
    }

    // Close the user file
    fclose(user_file);

    // Open the CAD file for reading
    FILE* cad_file = fopen("cad2023_2b_w8.txt", "r");
    if (cad_file == NULL) {
        perror("Error opening CAD file");
        return 1;
    }

    char unique_users[100][20]; // Assuming a maximum of 100 unique users with a length of 20 characters each
    int unique_user_count = 0;

    // Read the CAD file line by line
    while (fgets(line, sizeof(line), cad_file)) {
        char* token = strtok(line, " "); // Split the line by space
        if (token != NULL && strstr(token, "cad") == token) {
            // Extract the student number (skip "cad")
            char student_number[20]; // Assuming a maximum length of 20 characters for a student number
            strcpy(student_number, token + 3); // Skip "cad"
            
            // Check if the student number is in the list of valid users and not a duplicate
            int is_valid = 0;
            for (int i = 0; i < valid_user_count; i++) {
                if (strcmp(valid_users[i], student_number) == 0) {
                    is_valid = 1;
                    break;
                }
            }
            
            // Add the student number to the unique_users list if it's valid and not a duplicate
            if (is_valid) {
                int is_duplicate = 0;
                for (int i = 0; i < unique_user_count; i++) {
                    if (strcmp(unique_users[i], student_number) == 0) {
                        is_duplicate = 1;
                        break;
                    }
                }
                
                if (!is_duplicate) {
                    strcpy(unique_users[unique_user_count], student_number);
                    unique_user_count++;
                }
            }
        }
    }

    // Reverse the order of the unique student numbers
    for (int i = 0; i < unique_user_count / 2; i++) {
        char temp[20];
        strcpy(temp, unique_users[i]);
        strcpy(unique_users[i], unique_users[unique_user_count - 1 - i]);
        strcpy(unique_users[unique_user_count - 1 - i], temp);
    }

    // Print the unique student numbers in reverse order
    for (int i = 0; i < unique_user_count; i++) {
        printf("%s\n", unique_users[i]);
    }

    // Close the CAD file
    fclose(cad_file);

    return 0;
}

Network

這裡我們再來談與網路有關的議題, 其實就機械設計相關人員就使用網路的議題, 比起必須從底層設定網路的資訊工程相關人員來說, 已經非常單純與直覺. 但是為何課程中只要一提到與電腦軟硬體或網路相關的議題時, 多數人都會覺得使用上藏著許多秘訣. 其實不然. 因為機械設計工程師在運用電腦與網路, 最重要的議題就是:

能不能使用電腦與網路執行有效率的運算

以及

能不能運用電腦與網路進行協同設計

至於過程中的複雜度來自於:

  1. 幾乎所有的電腦軟硬體內定的回應都是英文
  2. 使用者對於自身長期使用的電腦與軟硬體相關配備與設置並非十分了解
  3. 電腦輔助設計室中若採用 IPv4 協定配置, 學員無法取得足夠上網頻寬
  4. 學校 IPv6 網路協定的設置並不完全
  5. github.com 至今(2023.12)仍只支援 IPv4

以下將逐一針對上列議題加以說明, 該如何因應:

熟悉電腦回應的英文訊息: 在使用電腦軟硬體過程, 當電腦回應各種英文訊息時, 儘量不要直接利用翻譯軟體轉為中文, 而應該設法了解每一個英文單字與術語的意思, 更好的習慣是, 在 B5 的紙本筆記本上謄寫各單字的英英字典查詢結果, 假以時日後, 將可充分了解各種情況下電腦的回應訊息, 並作出適當回應.

熟悉電腦軟硬體的配備: 無論使用自己或公用電腦時, 都必須設法查詢目前電腦軟硬體配備的規格, 以及所安裝的套件屬性, 針對可以取得的操作系統安裝檔案, 應該從空機自行安裝並設定網路. 其他常用的電腦套件也應該從該軟體的開發起源或目前所屬公司的相關資料了解起, 並自行安裝設定. 在充分了解電腦各式軟硬體安裝與配置的過程, 可以利用 Virtualbox 安裝虛擬主機, 並從中逐一測試不同軟硬體搭配下的各種表現, 從單機的單一網路卡配置開始, 到多台虛擬主機在不同軟硬上彼此透過不同的網路設定連線時的使用操作演練, 將可以最方便取得的模擬環境, 逐步熟悉電腦軟硬體相關內容與配置方法.

IPv4 網路的限制: 電腦輔助設計室中的電腦若採用 IPv4 連線機制, 必須透過 NAT 派送內部 IP 位址後, 統一以 NAT 的一個外部 IPv4 位址連外, 如此各電腦所能取得的對外頻寬, 並不足以讓學員順利操作各種 ssh, sftp, https 以及 git 網路連線對外指令. 因此若希望依賴連外網路上課, 必須局部或全部電腦採純 IPv6 網路上網. 因此在電腦輔助設計室上網的設定要訣是: 假如 IPv4 網速無法達到使用要求, 可以採純 IPv6 網路設定上網, 但是各種協定連線則必須各自設定雙網路協定支援的代理主機. 如此才能順利對目前仍只支援 IPv4 的伺服器連線(主要標的為 github.com).

學校的 IPv6 環境設置並不完全: 2023 年中之前, 儘管學校並無正常的 IPv6 DNS 伺服器可用, 但因允許使用中華電信的 DNS 伺服器設定, 因此只要將 IPv6 網路協定, 設為 DHCP6 取得 IPv6 位址後, 再搭配 hinet 的 IPv6 DNS 設置就可正常以網站的符號名稱連外. 但是目前 (2023.12) DHCP6 僅給定 address, 並無正確的 gateway 與 DNS 設定. 因此必須要自行設定固定 IPv6 address 與 gateway 之外, 還必須自行設定系上的 DNS server. 才能正常使用 IPv6 網路協定連外. (2016 年曾經規劃的系上 IPv6 位址分配, 也從未施行, mde.nfu.edu.tw 與 eng.nfu.edu.tw 網域上的 Google Workspace for Education 也一直放在雲端)

github.com 只支援 IPv4: 這個我們沒有辦法改變, 只能使用 IPv4 連線, 或者在純 IPv6 網路環境下, 針對 https 與 ssh 對 github.com 連線時, 透過雙網路協定支援的代理主機進行連線.

實際操作練習: 假如使用的電腦硬體是 Intel i7 且記憶體 8G 以上, 可以安裝 https://www.virtualbox.org/wiki/Downloads, 並下載 Windows 10 64位元 22H2.4 (在校網或透過 VPN 下載)開始進行各種配置與設定練習.

#include <stdio.h>

int main() {
    // Open a file to write displacement and velocity data
    FILE *outputFile = fopen("motion_data.txt", "w");
    if (!outputFile) {
        fprintf(stderr, "Failed to create data file.\n");
        return 1;
    }

    // Simulate motion for 10 seconds and calculate displacement and velocity, while writing data to the file
    double x = 0.2;  // Initial displacement
    double v = 0.0;  // Initial velocity
    double dt = 0.01; // Time step
    double t = 0.0;  // Time

    while (t <= 10.0) {
        double acceleration = (-10.0 * x - 0.5 * v) / 1.0; // Modified system parameters here
        v += acceleration * dt;
        x += v * dt;

        fprintf(outputFile, "%lf %lf %lf\n", t, x, v);

        t += dt;
    }

    // Close the data file
    fclose(outputFile);

    // Start a Gnuplot process using popen
    FILE *gnuplotPipe = popen("gnuplot -persistent", "w");
    if (!gnuplotPipe) {
        fprintf(stderr, "Failed to start Gnuplot.\n");
        return 1;
    }

    // Use Gnuplot plotting commands, specify font and output as PNG
    fprintf(gnuplotPipe, "set terminal pngcairo enhanced font 'default,10' size 800,400\n");
    fprintf(gnuplotPipe, "set output 'motion_plot.png'\n");
    fprintf(gnuplotPipe, "set title 'Displacement and Velocity vs. Time'\n");
    fprintf(gnuplotPipe, "set xlabel 'Time (s)'\n");
    fprintf(gnuplotPipe, "set ylabel 'Displacement (m)'\n");
    fprintf(gnuplotPipe, "plot 'motion_data.txt' using 1:2 with lines lw 2 title 'Displacement', \
                             'motion_data.txt' using 1:3 with lines lw 2 title 'Velocity'\n");

    // Close the Gnuplot process
    fprintf(gnuplotPipe, "exit\n");
    pclose(gnuplotPipe);

    return 0;
}

Euler's Method(教學影片, 說明1, 說明2 in English):

https://tutorial.math.lamar.edu/classes/de/eulersmethod.aspx

針對常微分方程式: \(\frac{dy}{dx}=f(x,y)\)

假如 \(x_{n+1}\) 與 \(x_{n}\) 之間的距離非常接近, 則 \(\frac{y_{n+1}-y_{n}}{x_{n+1}-x_{n}}\approx f(x_{n},y_{n})\)

若取 \(h=x_{n+1}-x_{n}\) 作為 x 座標在每一次計算過程的增量值.

則可以在已知此一常微分方程式的起始值之後, 逐一透過 h 增量值計算此常微分方程式在各 x 座標點上的對應值:

\(y_{n+1}-y_{n}\approx f(x_{n},y_{n})h\)

// 包含標準輸出入程式庫的標頭文件
#include <stdio.h>

// 主函式
int main() {
    // Open a file to write displacement and velocity data
    FILE *outputFile = fopen("motion_data.txt", "w");
    if (!outputFile) {
        fprintf(stderr, "Failed to create data file.\n");
        return 1;
    }

    // Simulate motion for 10 seconds and calculate displacement and velocity, while writing data to the file
    double x = 0.2;  // Initial displacement
    double v = 0.0;  // Initial velocity
    double dt = 0.01; // Time step
    double t = 0.0;  // Time

    while (t <= 10.0) {
        double acceleration = (-10.0 * x - 0.5 * v) / 1.0; // Modified system parameters here
        v += acceleration * dt;
        x += v * dt;

        fprintf(outputFile, "%lf %lf %lf\n", t, x, v);

        t += dt;
    }

    // Close the data file
    fclose(outputFile);

    // Start a Gnuplot process using popen
    FILE *gnuplotPipe = popen("gnuplot -persistent", "w");
    if (!gnuplotPipe) {
        fprintf(stderr, "Failed to start Gnuplot.\n");
        return 1;
    }

    // Use Gnuplot plotting commands, specify font and output as PNG
    fprintf(gnuplotPipe, "set terminal pngcairo enhanced font 'default,10' size 800,400\n");
    fprintf(gnuplotPipe, "set output './../images/motion_plot.png'\n");
    fprintf(gnuplotPipe, "set title 'Displacement and Velocity vs. Time'\n");
    fprintf(gnuplotPipe, "set xlabel 'Time (s)'\n");
    fprintf(gnuplotPipe, "set ylabel 'Displacement (m)'\n");
    fprintf(gnuplotPipe, "plot 'motion_data.txt' using 1:2 with lines lw 2 title 'Displacement', \
                             'motion_data.txt' using 1:3 with lines lw 2 title 'Velocity'\n");

    // Close the Gnuplot process
    fprintf(gnuplotPipe, "exit\n");
    pclose(gnuplotPipe);

    return 0;
}

}
