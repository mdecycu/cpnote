C -> Javascript -> Brython -> Python -> Cython (C++) -> Rust or Mojo

C 目前除了用於數值分析運算外, 由於其跨硬體的可攜性優勢而經常被用於機電自動控制領域

將 C 改寫為 Javascript 後, 在網頁前端可以直接顯示結果, 也能與後端的各種運算軟硬體結合, 成為協同產品設計的有效工具: 例如: https://pmksplus.com/ 與 https://motiongen.io/ 都是典型的應用範例.

PMKS + 原始碼 (採 Typescript)

pmks+ 平面機構設計範例1, 機構合成範例1, 

然而目前 Javascript 與機械設計的結合性略低於 Python, 且近年來許多原本採 Matlab 執行運算與控制的應用, 已經可以完全透過 Python 程式取代, 因此若能在網頁前端使用 Javascript 與 Brython, 然後在網頁後端採用 Python + Cython, 除了可以在數值分析運算上以 C++ 的速度執行, 也可在後續導入許多與 Python 高度相關的 AI 套件, 例如: TensorFlow、Keras 與 PyTorch.

Matlab 打不過 Python, 於是推廣兩者一起使用. Matlab 與 Python 比較, 假如真的要比輸贏或者叫你不要再走回頭路了.

Brython 程式練習頁面: https://mde.tw/cpnote/content/Brython.html

https://mde.tw/cpnote/content/w10-w11.html 中的程式改以 Brython 編寫, 並將程式存入個人的 Github Gist 取用. How?

可攜系統中 home_ipv6 目錄中的 .gitconfig

[http]
	proxy = http://p4.cycu.org:3128
[user]
	name = 你的 github 帳號
	email = 登記在 github 帳號的 email
[safe]
	directory = *
[http "https://github.com"]
sslVerify = false
[pull]
	rebase = false

.gitconfig 設定說明:

[http]
    proxy = http://p4.cycu.org:3128

也可以透過命令列中輸入 git config --global http.proxy http://p4.cycu.org:3128 取得

這是設定 git 指令需要使用 http 或 https 協定擷取 cmsimde 時, 假如網路只支援 IPv6, 就必須透過代理主機連線到目前只支援 IPv4 的 github.com

[user]
    name = 你的 github 帳號
    email = 登記在 github 帳號的 email

是在 git commit -m "提交訊息" 時綁定身分用

[safe]
    directory = *

則是宣告目前所使用的檔案目錄都是 safe

[http "https://github.com"]
    sslVerify = false

表示當連線到 github.com 時所使用的 key 不用登記在 Windows 操作系統中

[pull]
    rebase = false

表示 git pull 時, 並不使用 rebase, 而是直接 fetch 遠端資料, 並且進行 merge 合併
